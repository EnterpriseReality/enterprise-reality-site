# ER-WEB-005 Asset Governance

## Purpose

This directory will govern screenshots, logos, SVGs, PNGs, exported figures and publication artwork used by the Enterprise Reality Website Engineering Handbook. No assets are added during Milestone 1.

## Naming Rules

Assets should use descriptive, stable names:

```text
ER-WEB-005-ASSET-<chapter-number>-<short-name>.<extension>
```

Derived exports should preserve the canonical source stem where applicable.

## Source Attribution and Licensing

Every third-party or externally derived asset must include source attribution and licensing evidence before publication. Unlicensed or unclear assets are prohibited.

## Screenshots

Screenshots must be public-safe, current, reviewed and redacted where necessary. They must not expose private paths, credentials, local secrets, unpublished drafts, personal data, employer/client information or private tooling context.

## Logos

Logos may be included only when the usage is approved, properly attributed where required and consistent with public publication policy.

## SVGs, PNGs and Exported Figures

Editable source files are canonical where an asset has a source format. PNG and other raster exports are derived unless explicitly approved as canonical artwork.

## Publication Artwork

Publication artwork must support understanding and must not imply capability, maturity, adoption or endorsement beyond approved public evidence.

## Redaction

Redaction must happen before an asset enters this directory. Do not rely on display cropping or CSS to hide sensitive information.

## Accessibility

Assets must have appropriate alternative text when referenced. Text embedded in images should be avoided unless a text equivalent is provided.

## Resolution

Raster assets must be high enough resolution for the intended display while avoiding unnecessary repository weight.

## Public/Private Review

Every asset requires public/private review before publication. Assets copied from private workspaces, private repositories, local machines or review sessions must be checked for sensitive content before commit.

## Font Files

Font-file publication is prohibited in this directory unless a future approved publication explicitly changes this rule. Use website-approved font delivery mechanisms instead.
